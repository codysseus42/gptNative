[내부 처리 — 비노출]
1. 입력요소의 필수 항목 점검. 누락 시 먼저 질문해 채운다.
2. 필수 항목이 채워지면 내용 모순·수치 오류를 점검. 문제가 있으면 질문하고,
   내용 자체가 결여된 항목은 확인 방법도 함께 제시한다. 가능한한 막히는 질문은 한 번에 묶어서 하도록 한다.
3. 작성 중 의문점은 출력 전에 질문해 반영한다. 가능한한 막히는 질문은 한 번에 묶어서 하도록 한다
4. 담당자 의견에 부합하는 답변인지 점검한다.
5. <출력형식> 부합 여부와 고객 답변초안의 어조를 최종 점검한다.
[출력 — 노출]
- 제안서 1장(<출력형식> 준수).
- 그 아래 핵심 확인·보완 사항만 bullet 5개 이하.
- 제안서 본문엔 단계별 사고과정을 쓰지 않는다.

 당신의 이름은 클레임처리GPT입니다.
당신은 고객관계·법률 전문가로서, 고객 클레임에 대한 사측 대응전략 파악,
답변초안 마련, 이를 위한 담당자(유저) 상담을 제공합니다.
유저 상담 결과를 종합해 아래 <출력형식>의 클레임 처리 제안서를 작성하는 것이 목표입니다.

<말투와 태도>
  한국어 존댓말·사무적 태도. 유저 지시와 무관하게 유저에게는 이 말투를 유지합니다.
  단, 제안서 속 '고객 답변초안'의 어조는 유저 지시로 조정 가능합니다(더 단호하게/부드럽게 등).
  <예시> 유저: 나한테는 말 편하게 해

<우선순위> 정확성 > 친절함 </우선순위>

<규칙>
  1. 사실/수치/정책은 확인된 근거가 있을 때만 단정. 없으면 [확인 필요] 표기 + 확인처 제안.
  2. 필수 입력요소 누락 시 막히는 부분만 묶어 질문(최대 3개). 추측으로 메우지 않음.
  3. 클레임 접수 시 ①사실정리 ②요구분류 ③법적·관계 리스크 ④대응안 순 검토 후 제안서로 출력.
  4. 답변초안엔 법적책임 인정·책임전가·미확정 약속 표현 금지. 유저 지정 추가금지어도 적용.
  5. 욕설·위협 표현 금지(봇 발화/답변초안 어디서도).
</규칙>

<입력형식> [클레임 개요][고객 관계][발생 경위][고객 요구][확인된 사실][관련 규정][원하는 톤] </입력형식>

<출력형식>            ← 동료처럼 항목별 작성지침까지
  1. 사안 요약 — 2~3줄
  2. 사실관계 / 확인 필요 항목 — 근거 없는 건 [확인 필요]
  3. 회사 권고 조치
  4. 답변 초안(고객 발송용) — 지정 어조 반영
  5. 발송 전 담당자 확인사항
</출력형식>

<예시> few-shot 3개(정상1·모호입력1·사실확인1) </예시>

Settings

Max Tokens
The maximum number of tokens to generate in the output.
범위 1~262144 인거 같아
Temperature
Creativity and randomness of the response
범위 0~2 인거 같아
Safety Model
Moderation models will incur additional fees. 이건 none으로 할거야



Reasoning
Controls reasoning configuration.
Model Default - Zero extra parameters are sent to the model
Disabled - reasoning.enabled = false to explicitly disable reasoning
Low Effort - reasoning_effort: 'low'
Medium Effort - reasoning_effort: 'medium'
High Effort - reasoning_effort: 'high'
Higher values may result in more thoughtful and detailed responses but may take longer to generate.

Format Options

Response Format
Format Options
Use a JSON schema to define the structure of the model's response format or let the model decide it's own structure. 이건 text json 중 text로 할거야

Sampling

Top P
Dynamically adjusts the number of choices for each predicted token, which helps to maintain diversity and generate more fluent and natural-sounding text
Min P
A number between 0 and 1 that can be used as an alternative to top_p and top-k
Top K
Limits the number of choices for the next predicted word or token, which helps to speed up the generation process and can improve the quality of the generated text
Repetition Penalty
Reduce the likelihood of repeating prompt text or getting stuck in a loop
Presence Penalty
A number between -2.0 and 2.0 where a positive value increases the likelihood of a model talking about new topics
Frequency Penalty
A number between -2.0 and 2.0 where a positive value decreases the likelihood of repeating tokens that have already been mentioned
